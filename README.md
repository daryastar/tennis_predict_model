# tennis_predict_model
Проект для написания модели, предсказывающей вероятность победы в теннисе 
Какие модели используются на данный момент: 
**ML models** 
**Elo Rating System:** Это статистическая система рейтингов, где каждому игроку присваивается рейтинг, отражающий его силу. Вероятность победы в матче рассчитывается как функция разницы рейтингов (например, если разница 100 баллов, ожидаемая вероятность ~64%). Рейтинг обновляется после матча на основе неожиданности результата (K-фактор регулирует изменения). Используется для предсказания исходов матчей на уровне турниров, с учетом поверхности (hard, clay, grass). Пример: В симуляциях турниров Elo точно предсказывает победителей, превосходя ATP-рейтинги в ~70% случаев.
**Glicko Model:** Bayesian-улучшение Elo, добавляющее параметр неопределенности (RD - rating deviation), который растет при неактивности игрока. Вероятность победы основана на нормальном распределении силы (θ ~ N(r, RD²)), с обновлением через MCMC или аналитические методы. Применяется к Grand Slam (2000-2019), где интегрируется с логрегрессией для предсказаний; точность ~73.4% для мужчин, лучше базового (72.4%) на основе рангов. Учет корта (clay-specific) повышает до 73.8%. Пример: Разница рейтингов >800 дает ~100% вероятность победы.
**Markov Chain Models:** Моделируют теннис как цепь Маркова, где состояния - текущий счет (очко, гейм, сет), а переходы - вероятности выигрыша очка (p на подаче, q на приеме). Предполагает независимость событий, позволяет симулировать весь матч stroke-by-stroke. Используется для анализа scoring system и предсказания исходов; например, в MATLAB для оценки amplification различий в силе. Пример: В симуляциях показывает, как momentum влияет на переходы, с предсказаниями на уровне очка/гейма (точность зависит от исторических p). Хорошо для insights в поведение, но требует данных о вероятностях.
**IID Point-Based Probabilistic Models:** Предполагают независимые и одинаково распределенные (IID) очки, где вероятность выигрыша очка на подаче (p) фиксирована. Строят иерархию: вероятность гейма G(p), сета S(p,q), матча M(p,q). Позволяет предсказывать на любом уровне (очко, гейм, сет, матч), оценивая метрики вроде total points. Ограничение - игнор momentum. Пример: В анализе матчей p=0.6 дает ~80% шансов на гейм; используется для симуляций, где слабые различия в p усиливаются в матче.
**Pairwise Comparison Models (Bradley-Terry):** Обобщение Elo, где сила игроков - коэффициенты в модели, вероятность победы A над B = E_a / (E_a + E_b). Обновляется на основе исторических парных сравнений, без детализации на очки. Используется для матчевых предсказаний с минимальными данными (win/loss). Пример: В теннисе учитывает surface-specific коэффициенты, достигая точности ~70% в турнирах; просто и интерпретируемо.
Эти методы полагаются на вероятностные и статистические подходы, часто используя исторические данные для калибровки вероятностей. Они проще ML, но могут быть менее точными в complex сценариях; для деталей изучи источники.
