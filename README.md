# tennis_predict_model
Проект для написания модели, предсказывающей вероятность победы в теннисе 
Модель теннис 

Термины: 
Momentum - психологический фактор в игре, зависящий от количества последних выигранных/проигранных очков. Кратко про суть: если игрок много раз подряд выиграл очко, то он чувствует себя увереннее и играет лучше. Это сильно аффектит исходы матчей. 

Вот самые популярные виды моделей, которые дают наилучший результат. 

**Elo Rating System**
Это рейтинговая система, в которой вероятность выигрыша рассчитывается как функция разности рейтингов. Обновляется после каждого матча, учитывает тип поверхности. Работает лучше, чем рейтинг ATP. 
Плюсы: 
1. Прост в реализации 
2. Быстро адаптируется (тк вычисляется после каждого матча)
Минусы: 
   1. Плохо справляется, если был долгий перерыв 
   2. Чувствителен к выбору единственного гиперпараметра 
   3. Игнорирует momentum 
Эта модель находит улучшение в **Glicko Model**, где мы добавляем еще один фактор - фактор неопределенности, который хорошо обрабатывает долгие перерывы. 

**Markov Chain Models**
Модель симулирует матч как цепь Маркова. Это значит, что в качестве входных данных она принимает текущий счет, сопоставляет дальнейшему проигрышу/выигрышу некоторые вероятности (свои для подачи и для приема) и таким образом моделирует весь матч очко за очком. 
Плюсы: 
1.  Высокая детализация, можно получить вероятность победы в любой единице матча
2. Можно сгенерить кучу различных сценариев 

Минусы: 
1. Сильно зависит от входных вероятностей -> ошибка быстро копится 
2. Классическая версия предполагает независимость очков -> плохо ловится momentum 
3. Нужна большая поочковая статистика 
4. Вычислительно затратно 

**ML**
Тут все понятно: собрали набор фичей, закинули в модель и на выходе получили предсказание. Этот метод будем рассматривать отдельно.
